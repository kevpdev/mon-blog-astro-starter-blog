# Template — Commande Standardisée

Template unifié pour toutes les commandes Claude dans le projet.

## Structure obligatoire

```markdown
# Commande — [Nom descriptif]

[Description courte de l'objectif en 1-2 phrases]

## Usage

```
/[nom-commande] [paramètres optionnels]
```

[Contexte d'utilisation si nécessaire]

## Actions

1. [Action 1 avec verbe d'action]
2. [Action 2 avec critères spécifiques] 
3. [Action 3 avec validation/confirmation]

## Exemples

**Entrée :**
```
[exemple d'input]
```

**Sortie attendue :**
```
[exemple d'output ou comportement]
```

## Notes

- [Contrainte ou règle importante]
- [Comportement spécial ou exception]
```

## Conventions de nommage

- **Fichier** : `nom-commande.md` (kebab-case)
- **Titre** : `# Commande — [Nom descriptif]` (format uniforme)
- **Usage** : `/nom-commande` (slash prefix obligatoire)

## Règles de contenu

### Description
- Maximum 2 phrases
- Objectif clair et mesurable
- Éviter le jargon technique

### Actions
- Verbes d'action (Ajouter, Générer, Analyser, Valider)
- Ordre logique d'exécution
- Critères de validation explicites

### Exemples
- Cas d'usage concret
- Format entrée/sortie si applicable
- Résultat attendu précis

### Notes
- Contraintes techniques
- Comportements spéciaux
- Prérequis ou dépendances

## Modèle de développement

1. **Copier ce template**
2. **Remplacer les placeholders [...]**
3. **Tester la commande en conditions réelles**
4. **Valider avec les standards du projet**
